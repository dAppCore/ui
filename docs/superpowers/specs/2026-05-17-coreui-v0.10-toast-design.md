## 1. Goal

Ship notification toast primitives: `<core-toast>` Shadow DOM bubble + `<core-toast-region>` container + `toast` programmatic helper. Standard seed: 4 severity levels with built-in SVG icons, 6 corner positions, auto-dismiss with pause-on-hover, sticky toasts (duration=0), action slot for buttons (Undo/Retry), dismiss button, slide+fade animations, full ARIA wiring (status vs alert per severity).

Independent notification tier ‚Äî no dependency on v0.5 / v0.6 / v0.7 / v0.8 / v0.3 / v0.4 / v0.9. Day-one useful for any app needing transient feedback.

---

## 2. Scope

### In scope (v0.10 "standard seed")

- `<core-toast>` ‚Äî Shadow DOM bubble with severity + duration + animation state machine + auto-dismiss timer + pause-on-hover
- `<core-toast-region>` ‚Äî Shadow DOM container with 6 corner positions, stacks toasts via flex with `column-reverse` for bottom regions
- `toast` helper ‚Äî namespaced API (`toast.show`, `toast.info`, `toast.success`, `toast.warning`, `toast.error`, `toast.dismiss`, `toast.dismissAll`, `toast.getRegion`) with singleton default region
- 4 severity levels: `info` / `success` / `warning` / `error` with built-in SVG icons + colour tints + ARIA role mapping
- 6 corner positions: `top-left` / `top-center` / `top-right` / `bottom-left` / `bottom-center` / `bottom-right` (default `top-right`)
- Auto-dismiss timer with pause-on-hover (mouseenter pauses with remaining time; mouseleave resumes)
- Sticky toasts: `duration="0"` disables auto-dismiss; manual via `toast.dismiss(id)` or close button
- Slide+fade animations with `data-state` state machine (closed ‚Üí opening ‚Üí open ‚Üí closing ‚Üí closed); region sets `--core-toast-slide-from` based on position
- Stack reordering animation (CSS `transition: transform` on each toast smoothly slides survivors into new positions when middle dismissed)
- Dismiss button (√ó) on every toast with `aria-label="Dismiss notification"`
- Action slot for trailing buttons (Undo/Retry) ‚Äî does not auto-dismiss the toast
- ARIA: `role="status"` (polite) for info/success, `role="alert"` (assertive) for warning/error; `role="region"` + `aria-label="Notifications"` on regions
- Built-in SVG icons per severity, override via `<svg slot="icon">‚Ä¶</svg>`
- AX-rule example tests, integration tests, README + CLAUDE.md docs

### Out of scope (deferred to v0.10.1)

- **Swipe-to-dismiss** on touch devices
- **Promise-based toasts** ‚Äî react-hot-toast pattern: `toast.promise(asyncFn, { loading, success, error })`
- **Max-stack limit** with collapse-to-counter ("+3 more notifications")
- **Auto-grouping** by content / type (collapse duplicates)
- **Custom themes** beyond CSS-variable overrides
- **Region transitions** (slide region into view; currently `position: fixed` is always visible)
- **Cancellable `core-toast-open`** (only `core-toast-close` is cancellable in v0.10)
- **Toast bubble with rich content** (consumer can slot anything, but no built-in formatting beyond message + action)

## 3. Architecture

### Layer position

v0.10 occupies its own "notification" tier. Sits on v0.2 foundation (`tokens`). Independent of all prior interactive tiers. Helper composes a singleton region in `document.body` lazily on first call.

### Element + helper model

```
LitElement
‚îú‚îÄ‚îÄ CoreToast (Shadow DOM)
‚îÇ   Attrs (reflected):
‚îÇ     severity: 'info'|'success'|'warning'|'error'   (default 'info')
‚îÇ     duration: number                                (ms; default 5000; 0 = sticky)
‚îÇ     open: boolean                                   (drives visibility; usually set by helper)
‚îÇ     data-state: 'opening'|'open'|'closing'|'closed' (animation phase, set internally)
‚îÇ   Internal:
‚îÇ     _dismissTimer: number | null
‚îÇ     _remainingMs: number
‚îÇ     _timerStartedAt: number
‚îÇ   Behaviour:
‚îÇ     On connect with duration > 0: start auto-dismiss timer
‚îÇ     mouseenter: pause timer (capture remaining time)
‚îÇ     mouseleave: resume with remaining time
‚îÇ     close() advances state machine + fires core-toast-close + removes from DOM
‚îÇ     State machine driven by transitionend/animationend with fallback timer
‚îÇ
‚îú‚îÄ‚îÄ CoreToastRegion (Shadow DOM)
‚îÇ   Attrs (reflected):
‚îÇ     position: 6 corners                             (default 'top-right')
‚îÇ   Behaviour:
‚îÇ     position: fixed CSS with corner placement
‚îÇ     flex column (top regions) or column-reverse (bottom regions)
‚îÇ     MutationObserver tracks slotted <core-toast> children
‚îÇ     Methods: addToast, removeToast, clear, toasts/toastCount getters
‚îÇ     Sets --core-toast-slide-from CSS var based on position (-20px for top, 20px for bottom)
‚îÇ
‚îî‚îÄ‚îÄ toast (programmatic helper ‚Äî module-level singleton)
    Internal:
      _defaultRegion: CoreToastRegion | null   (lazy-created on first call)
      _toastUid: number                         (monotonic counter for IDs)
    Public methods:
      show(message, opts?): string  ‚Äî returns toast id
      info / success / warning / error (msg, opts?): string  ‚Äî severity shortcuts
      dismiss(id): void
      dismissAll(): void
      getRegion(): CoreToastRegion
```

### Why this split

- `<core-toast>` is the bubble ‚Äî Shadow DOM for chrome + animation isolation
- `<core-toast-region>` is the container ‚Äî Shadow DOM for position + stack coordination
- `toast` helper is a JS function namespace ‚Äî most consumers reach for `toast.success('Saved!')` not `document.createElement('core-toast')`. Helper bridges declarative element API to imperative call site convenience.

Declarative usage (`<core-toast severity="info">‚Ä¶</core-toast>` inside an explicit `<core-toast-region>`) is fully supported for consumers who prefer markup.

## 4. Component APIs

### 4.1 `<core-toast>`

```html
<core-toast severity="success" duration="5000">
  File saved.
</core-toast>

<core-toast severity="error" duration="0">
  <svg slot="icon">‚Ä¶</svg>
  Network unavailable.
  <button slot="action">Retry</button>
</core-toast>
```

**Attributes (reflected):**
- `severity="info|success|warning|error"` (default `info`)
- `duration` (number, ms; default `5000`; `0` = sticky)
- `open` (boolean) ‚Äî drives visibility; set by helper or consumer
- `data-state="opening|open|closing|closed"` (set internally)

**Properties:**
- `id: string` ‚Äî auto-set by helper as `data-toast-id`

**Methods:**
- `show(): void` ‚Äî open the toast
- `close(): void` ‚Äî close (fires `core-toast-close`, animates out, removes from DOM)
- `pauseTimer(): void` ‚Äî pause auto-dismiss
- `resumeTimer(): void` ‚Äî resume with remaining time

**Events:**
- `core-toast-open` ‚Äî fires when state reaches `open`
- `core-toast-close` ‚Äî cancellable; fires when state reaches `closed`; `preventDefault()` keeps toast visible (reverts state to `open`)

**Slots:**
- default ‚Äî message text
- `icon` ‚Äî override built-in severity icon
- `action` ‚Äî trailing action button (Undo/Retry) ‚Äî clicking does NOT auto-dismiss

**Parts:**
- `toast` ‚Äî outer container
- `icon` ‚Äî icon wrapper
- `message` ‚Äî message text wrapper
- `action` ‚Äî action slot wrapper
- `close-button` ‚Äî dismiss √ó button

**ARIA wiring (set automatically by element):**
- `role="status"` for `severity="info|success"` (polite, screen reader announces after current speech)
- `role="alert"` for `severity="warning|error"` (assertive, interrupts)
- Close button: `aria-label="Dismiss notification"`

### 4.2 `<core-toast-region>`

```html
<core-toast-region position="bottom-center">
  <core-toast severity="info">Hello</core-toast>
</core-toast-region>
```

Or programmatically via `toast.show()` (helper creates singleton with `position="top-right"`).

**Attributes (reflected):**
- `position="top-left|top-center|top-right|bottom-left|bottom-center|bottom-right"` (default `top-right`)

**Properties:**
- `toasts: CoreToast[]` (read-only getter)
- `toastCount: number` (read-only getter)

**Methods:**
- `addToast(toast: CoreToast): void` ‚Äî appends + sets `open=true`
- `removeToast(toast: CoreToast): void` ‚Äî calls `toast.close()` (which removes after animation)
- `clear(): void` ‚Äî close all toasts

**Slots:** default ‚Äî accepts `<core-toast>` children

**Parts:**
- `region` ‚Äî outer container (fixed positioned)

**CSS positioning:**

```css
:host {
  position: fixed;
  z-index: var(--core-overlay-z-index, 100);
  display: flex;
  flex-direction: column;
  gap: var(--core-toast-region-gap, 8px);
  pointer-events: none;  /* toasts re-enable */
  margin: var(--core-toast-region-margin, 16px);
}
:host([position="top-left"])     { top: 0; left: 0; --core-toast-slide-from: -20px; }
:host([position="top-center"])   { top: 0; left: 50%; transform: translateX(-50%); --core-toast-slide-from: -20px; }
:host([position="top-right"])    { top: 0; right: 0; --core-toast-slide-from: -20px; }
:host([position="bottom-left"])  { bottom: 0; left: 0; flex-direction: column-reverse; --core-toast-slide-from: 20px; }
:host([position="bottom-center"]) { bottom: 0; left: 50%; transform: translateX(-50%); flex-direction: column-reverse; --core-toast-slide-from: 20px; }
:host([position="bottom-right"]) { bottom: 0; right: 0; flex-direction: column-reverse; --core-toast-slide-from: 20px; }
```

Bottom regions use `column-reverse` so new toasts appear at the bottom (nearest viewport edge) and stack upward.

**ARIA:**
- `role="region"`
- `aria-label="Notifications"` (consumer can override via attribute)

### 4.3 `toast` helper

```ts
import { toast } from '@dappcore/ui/toast';

// Full options
const id: string = toast.show('Message', {
  severity?: 'info' | 'success' | 'warning' | 'error',
  duration?: number,
  region?: CoreToastRegion,
});

// Severity shortcuts
toast.info('Note: ‚Ä¶');
toast.success('Saved!');
toast.warning('Are you sure?');
toast.error('Failed.');

// Programmatic dismissal
toast.dismiss(id);
toast.dismissAll();

// Singleton region (creates if not exists)
const region = toast.getRegion();
```

**Internal state (module-level):**
- `_defaultRegion: CoreToastRegion | null` ‚Äî lazy-created on first call
- `_toastUid: number` ‚Äî monotonic counter for IDs

**Helper logic:**

```ts
interface ToastOptions {
  severity?: 'info' | 'success' | 'warning' | 'error';
  duration?: number;
  region?: CoreToastRegion;
}

interface ToastAPI {
  show(message: string, opts?: ToastOptions): string;
  info(message: string, opts?: Omit<ToastOptions, 'severity'>): string;
  success(message: string, opts?: Omit<ToastOptions, 'severity'>): string;
  warning(message: string, opts?: Omit<ToastOptions, 'severity'>): string;
  error(message: string, opts?: Omit<ToastOptions, 'severity'>): string;
  dismiss(id: string): void;
  dismissAll(): void;
  getRegion(): CoreToastRegion;
}

export const toast: ToastAPI;
```

**`getRegion()` behaviour:**
- If `_defaultRegion` is set AND still connected to DOM: return it
- Otherwise: create `<core-toast-region position="top-right">`, append to `document.body`, store as `_defaultRegion`, return it

Handles the case where consumer removes the region from DOM (e.g. test teardown) ‚Äî recreates on next call.

**`show()` behaviour:**
1. Resolve target region: `opts.region ?? getRegion()`
2. Create `<core-toast>` element
3. Set attributes: `severity`, `duration` (from opts or defaults)
4. Generate id: `core-toast-{++_toastUid}`
5. Set `data-toast-id="{id}"` on toast
6. Set message text content (`toast.textContent = message`)
7. Call `region.addToast(toast)` (which appends and sets `open=true`)
8. Return id

**`dismiss(id)` behaviour:**
1. Find toast: `_defaultRegion?.querySelector(`[data-toast-id="${id}"]`)`
2. If found: call `.close()` (no-op if not in singleton; for non-singleton regions consumer must hold a reference)

**`dismissAll()` behaviour:**
1. If `_defaultRegion` exists: call `_defaultRegion.clear()`

## 5. Behaviour details

### 5.1 Auto-dismiss timer + pause-on-hover

State per `<core-toast>`:
- `_dismissTimer: number | null` ‚Äî setTimeout ID
- `_remainingMs: number` ‚Äî time left when paused
- `_timerStartedAt: number` ‚Äî `Date.now()` when timer last started

**On `connectedCallback` (with `duration > 0`):**
1. `_remainingMs = this.duration`
2. `_startTimer()` ‚Äî sets `_timerStartedAt = Date.now()`; `_dismissTimer = setTimeout(() => this.close(), _remainingMs)`

**On `mouseenter` (host):**
1. If `_dismissTimer` null: no-op
2. `_remainingMs -= (Date.now() - _timerStartedAt)`
3. Clear timer, set `_dismissTimer = null`

**On `mouseleave` (host):**
1. If `_dismissTimer` null AND `_remainingMs > 0` AND `this.duration > 0`: `_startTimer()` with `_remainingMs`

**On `close()`:**
1. Clear `_dismissTimer` if set
2. Set `_dismissTimer = null`
3. Advance state machine `open ‚Üí closing` (fallback timer ensures advance under happy-dom)
4. After closing transition: `closing ‚Üí closed`, remove element from DOM

**`duration=0`:** never starts timer. Sticky until manual close.

### 5.2 State machine

```
closed ‚Üí opening ‚Üí open ‚Üí closing ‚Üí closed
```

| State | `open` attr | `data-state` | Visible? | Interactive? |
|---|---|---|---|---|
| `closed` | absent | `closed` | `display: none` | no |
| `opening` | present | `opening` | yes, transitioning in | yes |
| `open` | present | `open` | yes, at rest | yes (timer running) |
| `closing` | absent | `closing` | yes, transitioning out | no |

Driven by `transitionend` / `animationend` on host. **Fallback timer:** `--core-overlay-duration + 50ms` (happy-dom never fires transition events, same pattern as v0.8 surfaces).

`core-toast-close` is cancellable: when fired from `closing ‚Üí closed` advance, `preventDefault()` reverts to `open` state.

### 5.3 Animations

```css
:host {
  --duration: var(--core-overlay-duration, 200ms);
  --easing: var(--core-overlay-easing, cubic-bezier(0.4, 0, 0.2, 1));
}
:host([data-state="opening"]) {
  opacity: 0;
  transform: translateY(var(--core-toast-slide-from, -20px));
}
:host([data-state="open"]) {
  opacity: 1;
  transform: translateY(0);
  transition: opacity var(--duration) var(--easing),
              transform var(--duration) var(--easing);
}
:host([data-state="closing"]) {
  opacity: 0;
  transform: translateY(var(--core-toast-slide-from, -20px));
  pointer-events: none;
  transition: opacity var(--duration) var(--easing),
              transform var(--duration) var(--easing);
}
:host([data-state="closed"]) { display: none; }

@media (prefers-reduced-motion: reduce) {
  :host { transition: none !important; }
}
```

Region sets `--core-toast-slide-from` via `:host([position])` CSS rules (see ¬ß4.2).

**Stack reordering:** every `<core-toast>` has `transition: transform 200ms`. When a middle toast is removed, surviving toasts smoothly slide into their new flex positions. Pure CSS ‚Äî no JS.

### 5.4 Built-in severity icons

Four inline SVG icons rendered automatically based on `severity`:

- **info**: filled circle with "i"
- **success**: filled circle with check
- **warning**: filled triangle with "!"

All in `currentColor` so they tint with `--core-toast-fg`. Rendered inside `[part="icon"]` slot wrapper. Consumer override via `<svg slot="icon">‚Ä¶</svg>` ‚Äî slot fallback shows built-in when nothing slotted.

### 5.5 Singleton region lifecycle

`toast.getRegion()`:

```ts
function getRegion(): CoreToastRegion {
  if (_defaultRegion && _defaultRegion.isConnected) return _defaultRegion;
  _defaultRegion = document.createElement('core-toast-region') as CoreToastRegion;
  _defaultRegion.setAttribute('position', 'top-right');
  document.body.appendChild(_defaultRegion);
  return _defaultRegion;
}
```

**`isConnected` check** handles cases where consumer removes the region (test teardown, SPA navigation). Helper recreates on next call.

### 5.6 ID generation

```ts
let _toastUid = 0;
function nextToastId(): string {
  return `core-toast-${++_toastUid}`;
}
```

Set on toast as `data-toast-id`. `toast.dismiss(id)` does `region.querySelector(`[data-toast-id="${id}"]`)`.

### 5.7 Accessibility

| Severity | ARIA role | Live politeness |
|---|---|---|
| `info` | `status` | polite |
| `success` | `status` | polite |
| `warning` | `alert` | assertive |
| `error` | `alert` | assertive |

`role="status"` and `role="alert"` have implicit `aria-live` per ARIA spec ‚Äî no explicit `aria-live` needed.

Dismiss button: `aria-label="Dismiss notification"`.
Action slot: consumer responsible for content label.
Region: `role="region"` + `aria-label="Notifications"` (consumer can override).

**Reduced motion:** transitions zeroed via `@media (prefers-reduced-motion: reduce)`. State machine still advances via fallback timer.

### 5.8 happy-dom guardrails

- Custom element registration ‚úì
- `slotchange` ‚úì ‚Äî region uses for slot tracking
- `MutationObserver` ‚úì ‚Äî region backup for child mutations
- `transitionend` / `animationend` never fire ‚Äî fallback timer is the de facto path under tests (same v0.8 pattern)
- `setTimeout` works ‚úì ‚Äî auto-dismiss tests use shortened durations (e.g. `100ms`) + waits
- `mouseenter` / `mouseleave` events dispatched manually in tests
- `Date.now()` works ‚úì ‚Äî but actual elapsed times in tests are unreliable; pause/resume tests assert state changes (`_dismissTimer === null`) rather than precise remaining time
- `getBoundingClientRect()` returns zeros ‚Äî stack reordering visual not testable; assert layout via attribute presence

Every behaviour relying on real-browser semantics (visual slide, focus traversal across stacked toasts, swipe gestures) is gated with `if (...) return;` early-returns and commented `// Playwright sweep covers ‚Ä¶`.
