# CoreUI v0.5 — Primitives Layer (Design)

> *Status:* Approved (brainstorm 2026-05-16). *Authors:* Snider, Cladius. *Track:* components-first slice (v0.5 of the RFC roadmap).
> *Foundation:* Builds on v0.2 utils (`@dappcore/ui` tokens / colour / math / animation / dom / a11y / platform / brand). Spec at `docs/superpowers/specs/2026-05-16-coreui-utils-extraction-design.md`.

## 1. Mission

Land the load-bearing primitives layer of CoreUI as 11 brand-neutral Web Components. Every UI agent reaches for buttons, icons, pills, status dots, labels, toggles, and cards constantly; without these in the lib, every consumer reinvents them. This PR closes that gap.

Builds directly on the v0.2 utils foundation (tokens, colour, dom, a11y, platform, brand controllers). No new runtime dependencies beyond what v0.2 already added.

This is the **components-first slice** of the two-track plan. v0.3 (`<core-data-table>`), v0.7 (form components), and v0.8 (surface components) get their own brainstorms after this lands.

## 3. Locked design decisions

| Decision | Choice |
|---|---|
| Default styling delivery | **Light DOM + external CSS files** per primitive. Skin via plain attribute selectors + `--core-<name>-*` custom properties. |
| Icon API | **Registry + slot escape hatch + 12 default icons.** `registerIcon(name, svg)` for custom; `<core-icon><svg>…</svg></core-icon>` for one-offs; lib ships check, x, chevron-{up,down,left,right}, plus, minus, info, warning, danger, search. |
| Form participation | **ElementInternals on button + toggle.** `static formAssociated = true`; submit via `internals.form?.requestSubmit()`; checkbox value via `setFormValue`. |
| Execution shape | **Greenfield refactor.** Each primitive rewritten fresh against the v0.2 foundation. Source `lethean-*.ts` is a behaviour reference, not a starting point. |
| Pacing | **One bundled PR** with logical commit clusters (~13–15 commits). |
| Shadow DOM | Not used in v0.5. RFC §4 reserves Shadow DOM for v0.8 surface components needing backdrop isolation. |

## 5. Shared infrastructure

### 5.1 Light-DOM base class

```ts
// src/primitives/_shared/light-dom.ts
import { LitElement } from 'lit';

export class CoreElement extends LitElement {
  /** Render into light DOM so tokens cascade through and skins target plain selectors. */
  protected createRenderRoot(): HTMLElement {
    return this;
  }
}
```

Every primitive extends `CoreElement`. Light DOM means:
- Tokens (`--core-*`) cascade naturally; no Shadow-piercing needed.
- Skin layers target `core-button [part="base"]` (attribute selector) instead of `core-button::part(base)` (Shadow pseudo-element).
- Form participation via inner `<button>`/`<input>` elements works without ElementInternals plumbing — though button + toggle still adopt ElementInternals for custom validity in v0.7 readiness.

### 5.2 `part` vocabulary

**Clarification of the RFC's `::part()` contract under light DOM.** `::part()` is a Shadow-DOM-only pseudo-element selector. Our light-DOM primitives still attach `part="…"` attributes to internal elements — those attributes are the stable skinning hooks — but skin layers target them via standard attribute selectors (`core-button [part="base"]`), not `::part()`.

This is a deviation from the RFC's literal text. The RFC was written assuming Shadow DOM. Our light-DOM choice preserves the *intent* (stable skinning hooks; structural skinning without piercing encapsulation) via the more appropriate selector form.

Standardised vocabulary:

| `part` | Used by | Meaning |
|---|---|---|
| `base` | every primitive | The outer styled wrapper |
| `label` | button, pill, label, rail, toggle | User-visible text |
| `icon-leading` / `icon-trailing` | button, pill, rail | Slotted-or-inline icon |
| `indicator` | status-dot, toggle | State-bearing visual |
| `track` | toggle, sparkline | Linear background channel |
| `thumb` | toggle | Movable indicator over a track |
| `control` | window-controls | Each individual button (close/min/max) |
| `marker` | sparkline | Data point markers |

`_shared/parts.ts` documents these as TS string constants for IDE autocomplete, though primitives inline the strings in their templates (constants exist for documentation, not for runtime use).

### 5.3 CSS custom property schema

```
--core-<name>-<property>                  Component-level tunable
--core-<name>-<variant>-<property>        Variant-scoped tunable
--core-<name>-<property>-<state>          State-scoped tunable
```

Examples:

```css
--core-button-height: 32px;
--core-button-primary-bg: var(--core-brand-500);
--core-button-bg-hover: var(--core-ink-4);
--core-toggle-thumb-color-checked: var(--core-fg-0);
```

Resolution pattern (in default `.css` files):

```css
core-button { --core-button-bg: var(--core-ink-3); --core-button-fg: var(--core-fg-1); }
core-button[variant="primary"] { --core-button-bg: var(--core-brand-500); }
core-button[size="sm"] { --core-button-height: 24px; }
core-button:hover { --core-button-bg: var(--core-button-bg-hover, var(--core-ink-4)); }
core-button [part="base"] {
  height: var(--core-button-height);
  background: var(--core-button-bg);
  /* … */
}
```

Skin layers override variables (preferred) or selectors (escape hatch). Brand-mode flips cascade automatically because `--core-button-bg` resolves to `--core-brand-500`, which is re-bound by `[data-brand="…"]`.

### 5.4 Not in `_shared/`

- No `BaseInteractive` / `BaseFormControl` abstraction. Lit's reactive controllers + `ElementInternals` are enough.
- No CSS-in-JS helpers. Defaults live in `.css` files.
- No mixin for `part` decoration. Parts are inline attribute strings in templates.

## 6. Per-primitive surface

### 6.1 `<core-button>`

| Property | Detail |
|---|---|
| Attrs | `variant: primary | secondary | ghost | danger` (default `secondary`); `size: sm | md | lg` (default `md`); `disabled: boolean`; `type: button | submit | reset` (default `button`); `loading: boolean` |
| Slots | default (label), `slot="leading"`, `slot="trailing"` |
| Parts | `base`, `label`, `icon-leading`, `icon-trailing`, `spinner` |
| Form | `static formAssociated = true`; on click when `type="submit"`, calls `this._internals.form?.requestSubmit()` |
| A11y | Renders `<button type=…>` in light DOM; native button semantics apply. `aria-busy` reflects `loading`. `aria-disabled` mirrors `disabled`. |
| Events | Native click event (bubbles); no custom event needed |
| Vars | `--core-button-{height,padding-x,radius,bg,fg,border-color,font-size,bg-hover,bg-active}` plus variant overrides |

### 6.2 `<core-toggle>`

| Property | Detail |
|---|---|
| Attrs | `checked: boolean` (reflected); `disabled: boolean`; `name: string`; `value: string` (default `"on"`); `size: sm | md | lg` (default `md`) |
| Slots | default (label text) |
| Parts | `base`, `track`, `thumb`, `label` |
| Form | `static formAssociated = true`; calls `setFormValue(checked ? value : null)` on change |
| A11y | `role="switch"`, `aria-checked` reflected; Space/Enter toggle; Tab focus |
| Events | `core-change` (bubbles, composed); `detail: { checked: boolean }` |
| Vars | `--core-toggle-{height,track-color,track-color-checked,thumb-size,thumb-color,thumb-color-checked,radius}` |

### 6.3 `<core-status-dot>`

| Property | Detail |
|---|---|
| Attrs | `state: good | warn | bad | info | neutral` (default `neutral`); `size: sm | md | lg` (default `md`); `pulse: boolean` |
| Slots | none |
| Parts | `base`, `indicator` |
| A11y | `role="status"`; respects `aria-label` passed by consumer |
| Vars | `--core-status-dot-{size,fill,pulse-duration}`. `state` attribute selectors set `--core-status-dot-fill` to the appropriate state colour. |

### 6.4 `<core-pill>`

| Property | Detail |
|---|---|
| Attrs | `state: brand | neutral | success | warning | danger | info` (default `neutral`); `size: sm | md` (default `md`) |
| Slots | default (text), `slot="leading"`, `slot="trailing"` |
| Parts | `base`, `label`, `icon-leading`, `icon-trailing` |
| Vars | `--core-pill-{height,padding-x,radius,bg,fg,border-color,font-size}` plus state overrides |

### 6.5 `<core-icon>`

| Property | Detail |
|---|---|
| Attrs | `name: string` (registered icon key); `size: xs | sm | md | lg | xl | <length>` (default `md`); `decorative: boolean` (sets `aria-hidden`) |
| Slots | default — escape hatch for custom SVG: `<core-icon><svg>…</svg></core-icon>` |
| Parts | `base` |
| Registry | See §7 |
| A11y | If `decorative`, `aria-hidden="true"`. Otherwise: `aria-label` provided by consumer, falling back to registry entry's `title`. |
| Vars | `--core-icon-{size,color}`. Default `color: currentColor`. |

### 6.6 `<core-label>`

| Property | Detail |
|---|---|
| Attrs | `for: string` (mirrors HTML `<label for="…">`); `required: boolean` (adds `*` indicator); `size: sm | md | lg` (default `md`) |
| Slots | default (label text) |
| Parts | `base`, `required-indicator` |
| A11y | Renders a real `<label>` element with `for` mirrored. Native HTML labelling applies. |
| Vars | `--core-label-{font-size,fg,gap,required-color}` |

### 6.7 `<core-card>`

| Property | Detail |
|---|---|
| Attrs | `elevation: flat | raised | floating` (default `flat`); `padding: none | sm | md | lg` (default `md`); `interactive: boolean` |
| Slots | default, `slot="header"`, `slot="footer"`, `slot="media"` |
| Parts | `base`, `header`, `body`, `footer`, `media` |
| A11y | When `interactive`, sets `tabindex="0"` and emits a click event on Enter/Space. If a default-slot child is an `<a>` or `<button>`, that takes precedence (no synthetic event). |
| Vars | `--core-card-{bg,border-color,radius,padding,shadow,bg-hover}` + state overrides |

### 6.8 `<core-glass>`

| Property | Detail |
|---|---|
| Attrs | `dark: boolean` (switches tint); `radius: <length>` (default `var(--core-radius-lg)`) |
| Slots | default |
| Parts | `base`, `layer-blur`, `layer-tint` |
| CSS | `backdrop-filter: blur(...)` + tinted overlay. `@supports not (backdrop-filter: blur(1px))` falls back to opaque `--core-glass-tint-fallback` background. |
| Vars | `--core-glass-{radius,blur,tint,tint-dark,tint-fallback,border-color}` |

### 6.9 `<core-window-controls>`

| Property | Detail |
|---|---|
| Attrs | `platform: macos | windows | linux` (defaults to detected via `platform/getPlatform()`); `state: active | inactive` (default `active`) |
| Slots | none |
| Parts | `base`, `control` (each button); plus `[part="control"][data-action="close|minimise|maximise"]` |
| Events | `core-window-close`, `core-window-minimise`, `core-window-maximise` (composed, bubbles). Consumer wires to Wails runtime or platform-equivalent window manager. |
| Variants | macOS: red/yellow/green circles top-left. Windows: thin square icons top-right. Linux: GNOME-style top-right (consumer may override CSS for KDE / others). |
| Mobile / unknown platforms | If `getPlatform()` returns `ios`, `android`, or `unknown`, the element renders nothing visible (`display: none` via CSS). Consumer can still force a variant by setting `platform` attribute explicitly. |
| Vars | `--core-window-controls-{spacing,size,color-close,color-minimise,color-maximise,hover-darken}` |

### 6.10 `<core-rail>`

| Property | Detail |
|---|---|
| Attrs | `active: boolean`; `disabled: boolean`; `href: string` (anchor mode); `to: string` (router push mode) |
| Slots | default (label), `slot="leading"`, `slot="trailing"` |
| Parts | `base`, `label`, `icon-leading`, `icon-trailing`, `badge` |
| Events | `core-rail-navigate` (composed, bubbles) when `to` is set and the rail is **activated** (click or Enter/Space keypress); consumer wires to router. When `href` is set instead, native anchor navigation applies — no custom event. |
| A11y | Renders `<a>` when `href` set, `<button>` otherwise. Keyboard: Enter/Space activates. |
| Vars | `--core-rail-{height,padding-x,radius,bg,bg-active,bg-hover,fg,fg-active}` |

### 6.11 `<core-sparkline>`

| Property | Detail |
|---|---|
| Attrs | `points: string` (comma-separated numbers, e.g. `"1,3,2,5,4"`); `kind: line | area | bars` (default `line`); `width: <length>` (default `100%`); `height: <length>` (default `32px`); `stroke: <colour>`; `fill: <colour>` |
| Slots | default — alternative to `points`: `<data value="3"></data>` children. Attribute wins if both present. |
| Parts | `base`, `track`, `line`, `area`, `marker` |
| Implementation | Renders inline SVG; auto-fits to width/height. No axes, no tooltips, no interactivity — true sparkline. |
| Vars | `--core-sparkline-{stroke,stroke-width,fill,marker-size,track-color}` |

## 7. Icon registry

### 7.1 API

```ts
// src/primitives/icons/registry.ts
export interface IconEntry {
  svg: string;          // raw <svg>…</svg> markup
  title?: string;       // accessible title; used as aria-label fallback when not decorative
}

export function registerIcon(name: string, entry: string | IconEntry): void;
export function getIcon(name: string): IconEntry | null;
export function listIcons(): string[];
export function unregisterIcon(name: string): boolean;
```

- `registerIcon` accepts either raw SVG string or full `IconEntry`.
- Last write wins (consumers may override defaults).
- `<core-icon>` looks up via `getIcon` on `connectedCallback` and on `name` change.
- Missing name: logs `console.warn` once per name (deduped via internal `Set`); renders nothing; never throws.

### 7.2 Default icon set

Twelve icons registered as a side-effect of importing `src/primitives/icon.ts`:

| Name | Notes |
|---|---|
| `check` | Confirmation tick |
| `x` | Close / dismiss |
| `chevron-up`, `chevron-down`, `chevron-left`, `chevron-right` | Disclosure / navigation |
| `plus`, `minus` | Add / remove |
| `info`, `warning`, `danger` | State indicators (paired with status colours) |
| `search` | Search affordance |

All 12 are 16×16 viewBox, `currentColor` stroke, `stroke-width="1.5"`, `fill="none"` where applicable. Single visual style across the default set.

### 7.3 Slot escape hatch

```html
<core-icon><svg viewBox="0 0 24 24"><path d="…" /></svg></core-icon>
```

When the default slot has SVG content, the registered icon (if any) is ignored. Slot wins.

### 7.4 Tree-shaking note

Default icons register eagerly when `icon.ts` is imported. Consumers who want a smaller bundle and only their own icons can import just the registry (`@dappcore/ui/primitives/icons/registry`) — but they still need `icon.ts` for the `<core-icon>` custom element registration, so the saving is small. We document the two paths but don't optimise for the rare case.

## 8. Testing

Same discipline as v0.2:

- **`<primitive>.test.ts`** — happy-dom behaviour tests. Standard cases per primitive:
  - **Leaf primitives** (status-dot, pill, icon, label, card, glass, sparkline): renders without throwing, reflects key attrs to DOM, `part` attributes attached to the right elements, ARIA correctness.
  - **Interactive primitives** (button, toggle, rail, window-controls): all the above, plus click/keyboard behaviour, ARIA state correctness, ElementInternals form association for button and toggle.
- **`primitives_example_test.ts`** — one runnable AX example per primitive, copy-pastable into consumer code. ~11 examples.
- **Coverage:** ≥80% lines per primitive (matches v0.2 acceptance).

Specific test cases:

- `<core-button type="submit">` inside `<form>` submits the form (verify via `form.addEventListener('submit', handler)`).
- `<core-toggle>` `setFormValue` appears in `FormData` after submit.
- `<core-icon name="missing">` logs once, renders empty, doesn't throw.
- `<core-icon>` slot wins over `name` attribute.
- `<core-window-controls>` defaults to detected platform; explicit `platform="…"` overrides.
- `<core-sparkline points="…">` renders SVG; `<data>` children alternative path.

## 12. Risk register

| Risk | Mitigation |
|---|---|
| `::part()` Shadow-DOM-only confusion vs our light-DOM choice | Spec §5.2 documents explicitly; `part` attribute is the stable hook; skin selectors use attribute syntax not `::part()` pseudo |
| `ElementInternals` browser support | Shipping in all evergreens since 2023; document minimum browser baseline in README |
| Default icons conflict with consumer registrations | `registerIcon` is last-write-wins; documented explicitly in §7.1 |
| `<core-window-controls>` Linux DE flavour ambiguity | Ship GNOME-style by default; document `platform="linux"` attribute + CSS override path |
| `<core-sparkline>` performance with very large point sets | Document recommended max ~200 points; larger needs a chart lib |
| Light-DOM children of `<core-card>` losing slot semantics | Use `slot="…"` attribute on direct children; document convention; provide AX examples |
| `<core-button>` form submit inside nested forms | Native HTML constraint (forms can't nest); document briefly; not our problem to solve |
